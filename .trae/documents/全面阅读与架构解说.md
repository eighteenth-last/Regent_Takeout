## 前置检查

* 已安装 Git，命令可用（Windows PowerShell）

* 具有对 `https://github.com/eighteenth-last/Regent_Takeout.git` 的推送权限（推荐使用 GitHub PAT/已登录 Git Credential Manager）

## 初始化与配置

* 在项目根目录 `r:\Code_Repository\Regent_Takeout` 执行：

  * 初始化仓库：`git init`

  * 配置用户信息：

    * `git config user.name "eighteenth-last"`

    * `git config user.email "eighteenthstuai@gmail.com"`

  * 设置默认分支：`git branch -M main`

## .gitignore（可选，建议）

* 添加适用于 Java/Maven/IDEA 的忽略规则，避免提交 `target/`、`.idea/`、临时文件等

* 规则示例：

  * `target/`

  * `.idea/`

  * `*.iml`

  * `*.log`

  * `/.vscode/`

  * `Images/`（如为运行产生或外部资源，按需忽略）

## 提交与远程

* 暂存与提交：

  * `git add .`

  * `git commit -m "瑞吉外卖项目已完成"`

* 添加远程：`git remote add origin https://github.com/eighteenth-last/Regent_Takeout.git`

* 首次推送：`git push -u origin main`

## 身份认证与问题处理

* 首次推送将触发浏览器/凭据管理器登录或要求 PAT

* 如遇权限错误：确认仓库是否存在、账户是否拥有写权限

* 如远程已存在不同默认分支：改用 `master` 或执行 `git push -u origin main:main`

## 交付与验证

* 完成后返回 `git remote -v` 与 `git log -n 3` 验证

* 访问仓库页面确认提交历史与分支
